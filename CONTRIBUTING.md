# Contributing

Thanks for your interest in contributing to the Substack Writing Assistant skill!

## How to Contribute

### Reporting Issues

- Open an issue describing the problem or suggestion
- Include steps to reproduce if reporting a bug
- Share example use cases for feature requests

### Submitting Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes to `SKILL.md`
4. Test the skill with Claude Code
5. Commit your changes (`git commit -m 'Add your feature'`)
6. Push to your branch (`git push origin feature/your-feature`)
7. Open a Pull Request

### Guidelines

**Keep the philosophy intact:**
- The skill should empower writers, not replace them
- Never add features that write content for the user
- Feedback should ask questions, not provide answers

**Skill design principles:**
- Each mode should have a clear, single purpose
- Output formats should be consistent and scannable
